# Fraud_Detection_System_NovaPay_Sec
This project contains EDA - Exploratory Data Analysis, and ML models for fraud detection.

NovaPay faces rapidly evolving fraud threats as it expands globally. This project builds a real time, scalable fraud detection system that adapts to new tactics, strengthens transaction security, and protects customers while supporting NovaPay’s expansion.

## Business Challenge
The challenges Novapay hopes to solve include:
• Outdated fraud detection methods — Static rules and manual reviews can’t keep up with real-time, high-volume cross-border transactions or rapidly evolving fraud tactics.
• Rising financial losses — Chargebacks, refunds, and compliance penalties are eroding profitability, while missed fraud increases operational strain.
• High false positive rates — Legitimate customers are being incorrectly flagged, damaging user experience and risking customer churn.
• Regulatory pressure — AML and KYC frameworks require transparent, explainable fraud decisions, and failure to comply leads to penalties and increased scrutiny.
• Highly imbalanced fraud data — With fraud representing less than 1% of transactions, current tools struggle to detect rare events, leaving identity theft, account takeover, and transaction laundering as persistent threats.

## Target Variable for the Prediction.
The Target variable is "is_Fraud".
